# Hint System Design

## Goal

Let players spend XP to see step-by-step solution hints before answering a question, creating a meaningful trade-off between getting help and leveling speed.

## Trigger

- A "💡 提示" button appears in `BattleQuestionPanel` above the answer grid while the question is unanswered.
- Each press reveals one step from `question.steps[]` and deducts XP.
- Button disappears after answering (feedback shows remaining steps for free).

## Mechanics

| Aspect | Value |
|--------|-------|
| Cost per step | 20 XP (configurable via `balanceConfig.hint.costPerStep`) |
| Steps source | `GeneratedQuestion.steps[]` (already generated by all question types) |
| Disable condition | `pExp < costPerStep` or all steps already revealed |
| Reset | `hintsRevealed` resets to 0 when a new question loads |

## State Changes

- **New state:** `hintsRevealed: number` in battle reducer (default 0, reset on new question)
- **Existing state modified:** `pExp` decremented by `costPerStep` per hint

## Data Flow

```
BattleQuestionPanel (UI)
  → "💡 提示" button visible when !answered && steps.length > hintsRevealed
  → onClick → onRequestHint() callback (from useBattleCore)
    → guard: pExp >= costPerStep
    → setPExp(pExp - costPerStep)
    → setHintsRevealed(prev + 1)
  → UI renders steps[0..hintsRevealed-1] above answer buttons
```

## UI Layout

```
┌─────────────────────────┐
│  Question Card          │
│  "3/4 + 1/3 = ?"       │
├─────────────────────────┤
│  [Revealed hints here]  │  ← steps[0..N] shown progressively
│  Step 1. LCD = 12       │
│  Step 2. 9/12 + 4/12    │
├─────────────────────────┤
│  💡 提示 (-20 XP)       │  ← hint button (or disabled/hidden)
├─────────────────────────┤
│  [A] 13/12  [B] 1/2     │  ← answer grid (unchanged)
│  [C] 7/12   [D] 5/7     │
└─────────────────────────┘
```

## Non-Goals

- No new component — extend existing BattleQuestionPanel
- No changes to question generators — steps already exist
- Post-answer feedback steps remain free
- No hint usage tracking/analytics
